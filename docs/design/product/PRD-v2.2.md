## 1. 背景与目标

### 1.1 业务背景
当前K12家庭教育场景中存在三大痛点：
- **家长规划耗时**：每天为孩子规划学习任务耗时耗力，缺乏科学方法
- **孩子执行困难**：孩子拖延、专注力差，需要家长时刻监督
- **家校信息断层**：老师临时布置作业无法及时同步到孩子执行端

市面上现有解决方案存在明显短板：纯番茄钟APP（如Forest）缺乏家长协同；家校类APP（如钉钉）缺乏儿童友好的计时体验。

### 1.2 用户问题
| 用户角色 | 痛点 | 当前解决方式 | 存在的问题 |
|----------|------|--------------|------------|
| 家长 | 每天规划学习任务耗时 | 手写计划表 | 无法动态调整，难以追踪执行 |
| 家长 | 临时作业无法及时同步 | 微信/电话通知 | 信息传递有延迟，孩子容易遗忘 |
| 孩子 | 缺乏时间观念，容易分心 | 家长口头提醒 | 依赖性强，无法培养自主习惯 |
| 孩子 | 任务完成缺乏正向反馈 | 家长口头表扬 | 反馈不及时，激励效果弱 |

### 1.3 产品目标
- **总体目标**：打造「家长轻松规划、孩子自主执行、AI智能辅助」的亲子时间管理工具
- **量化目标（6个月内）**：
  - 家长创建首个计划激活率 ≥ 40%
  - 孩子7日留存率 ≥ 30%
  - 孩子日均使用时长 ≥ 20分钟
  - 家长端日活/孩子端日活 ≥ 60%

### 1.4 成功指标
| 指标 | 基线 | 目标值 | 测量方式 |
|------|------|--------|----------|
| 计划完成率 | - | ≥ 70% | 每日完成任务数/计划任务数 |
| 番茄钟使用率 | - | ≥ 80% | 使用番茄钟的任务数/总计时任务数 |
| 家长满意度 | - | NPS ≥ 40 | 月度问卷调查 |
| 孩子主动启动率 | - | ≥ 50% | 孩子主动点击开始/总任务开始 |

## 2. 用户分析

### 2.1 目标用户画像

**家长用户（决策者/付费者）**
- **角色**：25-45岁，K12学生家长，以妈妈为主
- **场景**：晚饭后规划次日学习任务、收到老师临时作业、周末制定复习计划
- **诉求**：省时省力、科学规划、实时了解孩子学习状态
- **痛点**：每天花30分钟以上规划、孩子执行不到位、缺乏正向引导方法

**孩子用户（使用者）**
- **角色**：6-14岁，小学生至初中生
- **场景**：放学后按清单完成任务、自主管理学习时间
- **诉求**：清晰的任务指引、有趣的游戏化激励、家长的认可
- **痛点**：任务太多不知从何开始、容易分心、缺乏成就感

### 2.2 用户故事

**家长端用户故事：**
- 作为家长，在晚上8点收到老师微信布置的额外作业时，我希望一键推送到孩子端，以便孩子立即知道并执行
- 作为家长，在周末为孩子制定复习计划时，我希望AI根据目标自动分解任务，以便科学分配时间
- 作为家长，在孩子学习过程中，我希望实时查看进度，以便了解孩子状态而不打扰
- 作为家长，在孩子完成任务后，我希望获得智能评语推荐，以便给予恰当的正向反馈
- 作为家长，我希望查看孩子一段时间内的学习统计，以便了解孩子的进步情况

**孩子端用户故事：**
- 作为孩子，在放学回家后，我希望看到清晰的任务列表，以便知道今天要做什么
- 作为孩子，在开始一项作业时，我希望点击开始就能自动进入专注模式，以便不被打扰
- 作为孩子，在完成任务后，我希望看到星星增加和庆祝动画，以便获得成就感
- 作为孩子，在番茄钟休息结束时，我希望收到温柔的提醒，以便知道可以继续学习
- 作为孩子，我希望手动勾选打卡项完成，以便记录我的好习惯

## 3. 功能需求

### 3.1 功能概述
AI时间管理闹钟APP是一款单一应用支持家长/孩子双角色的时间管理工具。家长模式负责任务规划与监控，孩子模式负责任务执行与计时，通过角色切换在同一APP内无缝流转。核心特色是内置计时器与番茄钟的智能联动，以及AI辅助的任务推荐与计划生成。

**核心设计原则：**
- 单一应用：降低用户下载和使用门槛
- 角色隔离：孩子模式无法直接访问家长功能，需验证切换
- 横屏优化：孩子模式专为横屏设计（手机+平板自适应）
- 简单易用：功能入口扁平化，首页即核心，其他功能从主页面跳转
- 任务与打卡分离：计时任务和习惯打卡分开展示，清晰区分

### 3.2 用户流程图

**流程1：家长首次使用**
```
下载APP → 微信一键登录 → 创建家庭 → 生成邀请码/二维码
→ 孩子扫码绑定 → 设置孩子信息 → 进入今日任务管理看板
→ 通过计划管理创建任务/习惯/目标 → AI辅助生成任务列表 
→ 确认并推送至孩子端 → 实时监控孩子执行
```

**流程2：孩子日常学习（核心流程）**
```
打开孩子端-横屏展示 → 查看今日任务列表+打卡项 → 选择任务
→ 点击开始计时 → 番茄钟自动启动25分钟倒计时 → 专注学习
→ 25分钟到 → 自动暂停任务计时 → 语音播报 → 进入5分钟休息
→ 休息结束 → 孩子手动继续 → 启动新番茄钟+继续任务计时
```

**流程3：番茄钟结束但任务未完成**
```
番茄钟25分钟结束 → 自动暂停任务计时
→ 语音播报:"番茄钟结束了，任务计时已暂停，但任务还没有完成哦"
→ 显示继续任务按钮 → 孩子点击继续 → 启动新番茄钟+继续任务计时
```

**流程4：打卡项完成流程**
```
孩子查看打卡项列表 → 点击打卡项复选框 → 切换完成/未完成状态
→ 完成时播放小动画
```

**流程5：临时作业推送（通过计划管理-今日任务添加）**
```
家长收到老师作业 → 进入今日任务管理看板 → 点击"添加任务"
→ 输入作业描述 → AI识别任务类型+预估时长 → 家长确认/修改
→ 一键推送至孩子端 → 孩子端实时通知 → 自动加入今日计划
```

**流程6：目标导向计划生成（通过计划管理）**
```
家长进入计划管理 → 设置长期目标 → AI分析目标 → 推荐阶段性任务分解
→ 选择场景模板 → 批量生成周计划/月计划（日历视图）
→ 家长修改确认 → 推送至孩子端执行 → 系统追踪目标进度
```

**流程7：超额完成提醒家长**
```
孩子超额完成任务 → 检测超额类型 → 推送消息给家长
→ 附带表扬话术参考 → 家长收到提醒
```

**流程8：每周总结与评比**
```
周末自动生成周报 → 统计本周数据（习惯达标率/任务完成率）
→ 计算星星奖励 → 效率时段分析 → 科目效率分析
→ 找出闪光点 → 生成科学点评 → 推送给家长和孩子
```

**流程9：每日总结推送**
```
孩子完成所有任务 → 自动生成今日总结 → 推送给家长
→ 附多条鼓励评语供选择 → 家长选择发送 → 孩子端显示在心语墙
```

### 3.3 功能详细说明

#### 3.3.1 角色切换与多用户模式（基础功能）

**功能描述**
单一APP支持家长和孩子的双角色使用，通过角色切换实现不同功能入口。支持同一设备上多孩子账号切换。

**设置页面归属**
- **家长端**：包含所有设置功能（账号管理、孩子信息管理、通用设置、通知设置、作息设置等）
- **孩子端**：仅保留护眼/休息相关基础设置（护眼模式开关、休息提醒开关），**通用设置移除了孩子端**

**交互逻辑**

**首次启动流程：**
1. APP首次打开显示「角色选择页」
2. 显示两个大图标按钮：
   - 👨‍👩‍👧 我是家长（家长模式入口）
   - 🧒 我是孩子（孩子模式入口）
3. 选择后进入对应模式的登录/绑定流程

**角色切换流程：**
1. **从孩子模式切换到家长模式**：
   - 点击设置页的「切换到家长模式」
   - 弹出验证：家长密码/指纹/FaceID
   - 验证通过后切换至家长模式
2. **从家长模式切换到孩子模式**：
   - 点击首页"切换到孩子模式"或设置页快捷入口
   - 显示孩子列表（多孩家庭）
   - 选择孩子后切换（无需验证，家长主动切换）

**多孩子支持：**
- 家长模式可管理多个孩子档案
- 孩子模式首次登录需选择"我是哪个孩子"
- 切换孩子需在设置页操作，需家长验证

**默认启动记忆：**
- 记忆上次使用的角色，下次打开直接进入
- 设备维度记忆（不同设备可设置不同默认角色）

**边界条件**
- 单设备最多绑定5个孩子账号
- 家长密码支持4位数字或生物识别
- 孩子连续10次验证失败，锁定切换功能1小时

**异常处理**
- 忘记家长密码：通过手机短信/邮箱验证重置
- 生物识别失败3次：自动切换为密码输入
- 网络异常：本地验证缓存，联网后同步

#### 3.3.2 家长端-计划管理（核心模块 - 功能合并）

**功能定位**
将"创建计划/习惯管理/目标管理/临时作业"合并为统一的**"计划管理"**模块，作为家长端的核心功能入口。

**模块结构**
```
计划管理
├── 今日任务（首页看板默认页）
│   ├── 今日计时任务列表
│   ├── 今日打卡习惯列表
│   └── 快速添加任务（原临时作业入口）
├── 日历视图（新增）
│   ├── 月历总览
│   ├── 批量创建计划
│   └── 查看/修改历史计划
├── 场景模板
│   ├── 系统模板（日常学习/周末复习/寒暑假/考前冲刺）
│   └── 我的模板（自定义保存）
└── 目标设置
    ├── 长期目标（学期/半年）
    └── 短期目标（周/月）
```

##### 3.3.2.1 今日任务（首页看板）

**功能描述**
家长端首页展示孩子今日完整的任务列表与打卡列表，支持直观查看、新增、修改、状态设置等操作。

**页面布局**
```
┌─────────────────────────────────────┐
│  孩子状态栏：头像 | 状态(专注中)     │
├─────────────────────────────────────┤
│  今日统计：已完成 3/5 | 专注 45分钟  │
├─────────────────────────────────────┤
│  【计时任务】        【打卡习惯】    │
│  ┌──────────────┐   ┌──────────────┐│
│  │ [进行中]     │   │ [✓] 早起    ││
│  │ 数学口算     │   │ [ ] 阅读30分 ││
│  │ ⏱️ 15分钟    │   │ [✓] 整理书包││
│  └──────────────┘   └──────────────┘│
│  ┌──────────────┐                  │
│  │ [待开始]     │   [+ 添加打卡]   │
│  │ 语文课文背诵 │                  │
│  │ ⏱️ 20分钟    │                  │
│  └──────────────┘                  │
│                                    │
│      [+ 添加今日任务]              │
└─────────────────────────────────────┘
```

**交互逻辑**
1. **任务列表展示**：
   - 显示今日所有计时任务（学科图标、任务名称、预估时长、状态）
   - 状态标签：待开始/进行中/已完成/暂停
   - **进行中任务**：高亮显示，显示"专注中"标签，进度条展示
   - **当前专注中的任务**：不可编辑（修改按钮置灰），仅可查看

2. **打卡习惯展示**：
   - 显示今日所有打卡项
   - 显示完成状态（已完成/未完成）

## 4. 云端服务

### 4.1 AI智能引擎
基于阿里百炼大模型，实现：
- 任务描述智能解析（分类、预估时长）
- 评语推荐
- 周报生成
- 闪光点分析

### 4.2 实时通信服务
采用WebSocket保障家长端与孩子端之间的状态实时同步：
- 任务状态同步
- 番茄钟进度同步
- 远程推送

### 4.3 数据存储与API
管理用户信息、家庭关系、任务计划、番茄钟记录、成就数据等，提供RESTful API供客户端调用。

### 4.4 知识库管理
维护学科、教辅术语映射（如"《补》"→"语文补充习题"），支持家长自定义添加，提高AI解析准确性。

## 5. 非功能需求

### 5.1 性能要求
- 接口响应时间：平均 < 200ms，P99 < 500ms
- APP启动时间：冷启动 < 2秒
- 实时同步延迟：WebSocket消息 < 1秒
- 并发用户数：支持10万家庭同时在线

### 5.2 安全要求
- 数据传输：全链路HTTPS/TLS 1.3
- 用户隐私：儿童个人信息按《儿童个人信息网络保护规定》处理
- 权限控制：家长只能查看自己孩子的数据
- 敏感操作：修改绑定关系需二次验证

### 5.3 兼容性要求
- **孩子端**：Android 8.0+ / iOS 13+，支持手机和平板横屏自适应
- **家长端**：Android 7.0+ / iOS 12+
- **屏幕适配**：手机横屏最小宽度640dp，平板适配7-13英寸

### 5.4 可用性要求
- 离线模式：孩子端支持离线计时，联网后同步数据
- 容错机制：网络异常时本地缓存，自动重试
- 无障碍：支持系统字体大小设置
